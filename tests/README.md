# Test data

## Synthetic reference data (`data/synth_references.mat`)

Octave/MATLAB reference outputs for verifying the Python port against the
original `gridfit.m` (D'Errico, 2006). Generated by `generate_synth_references.m`.

### Input data

250 scattered points with realistic properties:

- **Irregularly spaced** — sampled from randn (clustered near center, sparse
  near corners), not on a regular grid
- **Non-uniform density** — dense in the middle, extra sparse corner points
  added explicitly
- **Measurement noise** — small Gaussian noise added to the true surface
- **Replicates** — ~20 duplicate (x,y) locations with slightly different z
- **Deterministic** — uses `rand('seed', 42)` for reproducibility

True surface: `z = sin(4x + 5y) * cos(7(x - y)) + exp(x + y)` on [0, 1]^2.

Fitted onto a 21x21 grid (`linspace(0, 1, 21)` for both axes), with
`smoothness=1` and `extend='never'`.

### Reference grids stored

21 output grids (each 21x21), keyed as `{interp}_{regularizer}_{solver}`:

| | gradient | diffusion | springs |
|---|---|---|---|
| **triangle** | normal, backslash | normal, backslash | normal, backslash |
| **bilinear** | normal, backslash | normal, backslash | normal, backslash |
| **nearest** | normal, backslash | normal, backslash | normal, backslash |

Plus 3 anisotropic smoothness `[1, 2]` grids (normal solver only):
- `aniso_triangle_gradient_normal`
- `aniso_bilinear_diffusion_normal`
- `aniso_nearest_springs_normal`

The Python port only implements `normal` and `lsqr` solvers. The `backslash`
grids are stored for cross-reference but are not tested against directly.

### Regenerating

Requires GNU Octave (or MATLAB) and `gridfit.m` from `dev/gridfitdir/`:

```bash
octave --no-gui --path dev/gridfitdir tests/generate_synth_references.m
```
